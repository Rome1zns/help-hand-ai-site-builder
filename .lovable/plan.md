

# Добавить бэкенд-логику в генерацию сайтов

## Что будет сделано
Сейчас ИИ генерирует только фронтенд (HTML/CSS/JS). После этих изменений ИИ сможет генерировать сайты с серверной логикой — хранение данных, формы обратной связи, списки задач и т.д. Данные будут реально сохраняться на сервере и не пропадут при перезагрузке.

## Как это работает
Сгенерированные сайты будут использовать облачную базу данных через JavaScript-библиотеку, подключаемую через CDN. ИИ будет знать, как записывать и читать данные, и будет автоматически добавлять эту логику в сгенерированный код.

---

## Технический план

### 1. Создать таблицу `user_data` (миграция)
Универсальное хранилище данных для сгенерированных сайтов:

```text
user_data
- id (uuid, PK)
- session_id (text) — привязка к сессии пользователя
- collection (text) — виртуальная "таблица" (например "contacts", "todos", "products")
- data (jsonb) — произвольные данные
- created_at (timestamptz)
```

RLS-политика: публичный доступ (сгенерированные сайты работают без авторизации).

### 2. Обновить системный промпт в `supabase/functions/generate-site/index.ts`
Расширить инструкцию для ИИ:
- Объяснить, что доступна база данных через Supabase JS CDN
- Указать URL проекта и anon-ключ (они публичные)
- Описать структуру таблицы `user_data` и примеры CRUD-операций
- Указать, что нужно использовать `session_id` для изоляции данных между пользователями
- Дать примеры: форма обратной связи, список задач, гостевая книга

### 3. Обновить агентов в `src/lib/agents.ts`
- Изменить статус-сообщение агента "Разработчик" — упомянуть бэкенд-логику
- Добавить в описание, что команда умеет писать и серверную часть

### 4. Обновить `src/components/editor/ChatPanel.tsx`
- В пустом состоянии чата обновить текст — указать что команда создаёт сайты с бэкендом

## Пример того, что ИИ сможет генерировать

Пользователь говорит: "Сделай список задач"

ИИ генерирует HTML с:
- Красивым интерфейсом списка задач
- JavaScript-кодом, который сохраняет/загружает задачи из базы данных
- Данные не пропадают при перезагрузке страницы
